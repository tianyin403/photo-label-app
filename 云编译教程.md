# 跨平台打包方案：无 Mac 如何编译 iOS IPA

> 适用场景：只有 Windows 电脑，没有 macOS
> 方案：使用免费云编译服务

---

## 方案一：Codemagic（推荐，最省心）

Codemagic 是专为 Flutter 设计的云编译平台，免费套餐每月 2000 分钟。

### 实施步骤

#### 1. 推送代码到 GitHub

```bash
# 在 GitHub 上新建一个仓库（私有/公开均可）
# 然后在本地推送
cd D:\project\codeing空间\摄影照片管理app\ios_app

# 初始化 git
git init
git add .
git commit -m "Initial commit: iOS photo label app"

# 推送到 GitHub（把 YOUR_USER 替换成你的 GitHub 用户名）
git remote add origin https://github.com/YOUR_USER/photo-label-app.git
git branch -M main
git push -u origin main
```

#### 2. 注册 Codemagic

- 访问 https://codemagic.io/
- 用 GitHub 账号登录
- 授权访问你的 photo-label-app 仓库

#### 3. 配置 Apple 开发者账号

> **免费账号** 也可以签名为个人使用的 app（手动信任安装）

1. 在 https://developer.apple.com 免费注册
2. 获取 Team ID：登录 Apple Developer → 账号页面 → 看 Membership
3. 在 Codemagic 中：Teams → Apple Developer Account → Add
   - 上传 P12 证书 + 描述文件
   - 或让 Codemagic 自动管理（推荐）

#### 4. 构建

Codemagic 检测到代码推送后会自动构建，也可以手动点击 "Start build"

#### 5. 下载 IPA

构建完成后，Artifacts 里就有 `.ipa` 文件，直接下载。

## 方案二：GitHub Actions（备选）

仓库已经包含了 `.github/workflows/ios_build.yml`，配置好以下 Secrets 即可：

| Secret | 说明 | 获取方式 |
|--------|------|---------|
| `APPLE_TEAM_ID` | Apple Developer Team ID | Apple Developer 后台查看 |

设置方法：GitHub 仓库 → Settings → Secrets and variables → Actions → New repository secret

然后在 GitHub 仓库页面点 Actions → 选 "Build iOS IPA" → Run workflow。

---

## 后续操作：安装到 iPhone

拿到 `.ipa` 文件后：

1. **安装 IPA 到手机**
   - Windows 上用 **爱思助手**：设备管理 → 应用 → 导入 IPA
   - 或用 **iMazing**（免费版即可）：拖入 IPA 文件
   - 也可以用 **AltStore** 安装

2. **手动信任证书**
   - iPhone 上：**设置 → 通用 → VPN与设备管理**
   - 找到你的开发者证书（显示为你的 Apple ID）
   - 点击 **"信任"**
   - 确认信任

3. **打开 App**
   - 返回桌面，点击 App 图标即可正常使用
